# Lab 1 - Отчет

## Выполненные задачи

### 1. Выбор фреймворка
Выбран Flask по следующим причинам:
- Легковесный и простой в изучении
- Минималистичный подход
- Идеально подходит для микросервисов
- Большое сообщество и документация

### 2. Реализованные эндпоинты

#### GET /
Возвращает:
- Информацию о сервисе (имя, версия, фреймворк)
- Информацию о системе (хостнейм, платформа, CPU)
- Время работы сервиса
- Информацию о запросе
- Список доступных эндпоинтов

#### GET /health
Возвращает:
- Статус сервиса (healthy)
- Текущее время
- Время работы в секундах

### 3. Конфигурация
Сервис настраивается через переменные окружения:
- `HOST` - хост для запуска (по умолчанию: 0.0.0.0)
- `PORT` - порт для запуска (по умолчанию: 5000)
- `DEBUG` - режим отладки (по умолчанию: False)

### 4. Лучшие практики
- Чистая структура кода
- Обработка ошибок (404)
- Использование виртуального окружения
- Фиксация версий зависимостей
- Документация в README.md
